EXPLICACIÓN DEL FUNCIONAMIENTO DEL WIDGET WISBE
==============================================

¿Cómo se muestra la data y el diseño en el sitio del cliente SIN ARCHIVOS adicionales?
------------------------------------------------------------------------------------

El sistema funciona mediante una arquitectura de "Inyección Centralizada". El cliente NO necesita tener nutricion.html, rutinas.html ni ningún otro archivo en su servidor. SOLO necesita el código:

   <div id="wisbe-nutricion" domain="gympro.xyz"></div>
   <script src="https://wisbe.xyz/widget.js"></script>

PROCESO INTERNO:

1. CARGA REMOTA: El script 'widget.js' se carga desde los servidores de Wisbe (wisbe.xyz). Este script contiene toda la "inteligencia" para construir el widget.

2. DETECCIÓN: El script busca en el sitio del cliente el <div> con el ID correspondiente (ej. wisbe-nutricion).

3. IDENTIFICACIÓN POR DOMINIO: El script lee el atributo 'domain="gympro.xyz"'. Este dominio es la CLAVE. No importa en qué sitio web esté pegado el código; el widget usará ese dominio para buscar en la base de datos de Wisbe qué recetas pertenecen a ese usuario específico.

4. DISEÑO "EMPAQUETADO": El script inyecta un <iframe> que carga la interfaz de Wisbe directamente desde wisbe.xyz.
   - El cliente ve los estilos y el diseño "Luxury" porque el iframe está cargando la versión alojada en Wisbe.
   - Todo el CSS de Tailwind, las fuentes y los íconos ya están dentro de esa página remota.

¿POR QUÉ ES MEJOR ASÍ?
----------------------

- Zero Mantenimiento para el Cliente: El cliente no tiene que actualizar archivos. Si tú cambias el diseño en wisbe.xyz, el sitio del cliente se actualiza AUTOMÁTICAMENTE.
- Portabilidad Total: El código funciona en WordPress, Shopify, Wix, o un sitio HTML puro.
- Seguridad: La lógica de conexión con Supabase se mantiene en Wisbe, no en el sitio del cliente.

TROUBLESHOOTING (Si no se ve nada):
----------------------------------

1. CONSOLA DEL NAVEGADOR: Presiona F12 y ve a "Console". Si el dominio no existe en Wisbe, verás un error indicándolo.
2. DOMINIO EXACTO: Asegúrate de que en el Administrador de Usuarios de Wisbe, el usuario tenga puesto EXACTAMENTE el mismo dominio que pusiste en el código (ej. si pusiste gympro.xyz, en el perfil no debe decir https://gympro.xyz).
3. IFRAME BLOCKING: Algunos compiladores de código online (como el de la imagen que enviaste) bloquean la carga de sitios externos por seguridad. Prueba el código en un archivo .html real en tu computadora.
